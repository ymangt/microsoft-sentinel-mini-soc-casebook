# Recruiter Review Checklist

Before publishing the repo, confirm that a skimming recruiter or hiring manager can see the value in under 60 seconds.

## First Impression

- Project name is clear.
- README says what the lab proves.
- Scenario table is near the top.
- Screenshots are visible and captioned.
- No giant unexplained code dump.

## Security Analyst Signal

- KQL detections are readable.
- Investigation queries are separate from detections.
- MITRE mapping is present but not exaggerated.
- False positives are discussed.
- Response recommendations are realistic for a junior analyst.

## Credibility

- The README explains which data is simulated.
- No claim implies production SOC experience.
- No unsafe activity is described.
- Cost and limitations are documented.

## Interview Readiness

- You can explain why each scenario matters.
- You can explain one detection line-by-line.
- You can discuss what would change in a production SOC.
- You can name one false positive and one tuning improvement.
